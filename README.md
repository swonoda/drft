# DRFT

**DRFT** は、書くこと・読むことに集中できる、Windows向けの軽量な小説執筆用エディタです。

執筆は入力しやすい横書きで。必要なときだけ縦書きプレビューを開き、読者に近い状態で文章を確認できます。ほかの執筆ツールを併用しやすいよう、原稿は単独のTXT、作品専用の辞書はMarkdownで管理します。

長い原稿でも軽快に動作します。手元では30万字のテキストまで入力遅延なく動作することを確認しています。

![DRFTの編集画面](docs/screenshot.png)

## 特徴

### 必要なときだけ開く縦書きプレビュー

執筆中はプレビューを閉じ、横書きのエディタだけで集中できます。

![エディタのみ](docs/editor.png)

プレビューを画面は余白・フォントなどを変更して好みの組版に仕上げることができます。

ページは左右ボタンで切り替わり、エディタのカーソルを含む一文を若草色で表示します。カーソルを動かすと該当ページへ自動で追従します。

![プレビュー画面](docs/preview.png)

> [!NOTE]
> 縦書き用でないフォントでは、プレビューの長音記号「ー」が横向きになることがあります。縦書きに対応したフォントを指定してください。

### 章と節を見渡せる構成ビュー

原稿へ専用の見出し記法を入れなくても、行空きから章と節を自動判別して一覧を作成します。一覧をクリックすると本文へジャンプでき、節はドラッグ＆ドロップで並べ替えられます。

- 一行空きで節、二行空きで章を判定
- 章の下へ節をネストして表示
- 節単位でドラッグ＆ドロップ
- 構成ビューはメニューから表示・非表示を切り替え

### `#fix` マークで修正箇所をメモ

本文中へ `#fix[コメント]` を置くと、修正ビューへ一覧表示されます。気になる箇所をエディタで管理できます。クリックすると本文の該当位置へ移動します。

修正後に本文からマークを消すと一覧からも消えます。マークが0件になればたぶん完成稿です。

![簡単にメモを残して管理](docs/fix_comment.png)

### 作品専用の辞書

登場人物、地名、ルビ、設定などを、見出しと説明だけのシンプルな辞書へ記録できます。辞書の見出しをクリックすると、原稿内の出現箇所を検索できます。

辞書は原稿と同じフォルダへ `原稿名_辞書.md` として保存されます。Markdown形式のテキストファイルなので、あとから設定集として利用したり、ほかのツールで編集したりするのも簡単です。

### 青空文庫記法・カクヨム記法に対応

ルビと傍点を縦書きプレビューとPDFへ反映します。傍点は縦書き向けのゴマで表示されます。

### 長い原稿でも軽快

小説は長いテキストファイルです。DRFTは入力を止めないことを優先し、入力遅延・クラッシュを最小限にするよう最適化しています。

- 30万字まで入力遅延なしを確認
- UTF-8／Shift_JISを自動判定し、画面下から保存形式を切り替え
- 本文文字数と400字詰め原稿用紙換算を常時表示

### 右綴じ見開きPDF

表示設定を反映したA5縦書きPDFを作り、右綴じの見開きへ面付けします。

## インストール方法

リリースページからダウンロードしたzipファイルを解凍します。

- インストールする場合：`DRFT Setup {バージョン番号}.exe`を実行
- インストールせずに使う場合：`DRFT {バージョン番号}.exe`を実行

## 基本的な使い方

1. 「ファイル」→「新規作成」または「開く」で原稿を用意します。
2. 横書きのエディタで本文を書きます。一度保存先を決めると、変更は自動保存されます。
3. 右上のプレビューアイコンから縦書き表示を開き、読み心地を確認します。
4. 必要に応じて「表示」→「辞書」を開き、人物名や地名、設定を記録します。
5. 完成後は「ファイル」→「PDF出力」から見開きPDFを作成します。

終了時に保存済みの原稿を開いていた場合、次回起動時に同じ原稿を復元します。辞書ウィンドウを開いたまま終了した場合は、辞書も再表示します。

## 原稿の構造

DRFTは空行から原稿の構造を読み取ります。

| 書き方          | DRFTでの扱い |
| --------------- | ------------ |
| 先頭の1行       | タイトル     |
| 1行空き         | 節の区切り   |
| 2行空きの次の行 | 章見出し     |
|                 |              |

文字数は、改行とルビ・傍点・修正マークの記法を除き、本文として表示される文字を数えます。全角空白と句読点はそれぞれ1字として扱い、400字で割った余りを切り上げて原稿用紙枚数へ換算します。

## 対応記法

現在対応している青空文庫記法・カクヨム記法は、ルビと傍点です。

### ルビ

```text
｜流氷《りゅうひょう》
|流氷《りゅうひょう》
流氷《りゅうひょう》
```

### 傍点

```text
重要［＃「重要」に傍点］
《《重要》》
```

### 修正マーク

```text
しろくまの姿が水平線のあたりにぼんやりと見える。#fix[季節が分かる描写を追加]
```

`[]`内のコメントは省略できます。修正マークはTXTにはそのまま保存されますが、文字数、縦書きプレビュー、PDFには表示されません。

## 原稿と辞書

辞書は、開いているTXTと同じフォルダの `原稿名_辞書.md` に保存されます。DRFTが管理しないファイルやサブフォルダは変更しません。

以前のバージョンで作成した `辞書.md` も読み込めますが、次の保存から原稿ごとの名前になります。

```text
作品名/
├── 作品名.txt
├── 作品名_辞書.md
├── その他のファイル
└── 資料/
```

辞書はMarkdownの見出しと説明で構成されます。

```markdown
# 主人公

物語の中心人物。好物は温かいスープ。

# 北港

物語の舞台となる架空の港町。読みはノースポート。
```

辞書の見出しをクリックすると、原稿内の同じ語へジャンプします。繰り返しクリックすると次の出現箇所へ進みます。変更後は保存ボタンまたは `Cmd/Ctrl+S` で保存できます。

## PDF出力

PDFはA5単ページを生成したあと、右綴じの見開きへ面付けします。本文先頭は論理2ページ目、つまり最初の見開きの左側へ配置され、右側は空白になります。

```text
見開き1  [ 2ページ ] [ 空白 ]
見開き2  [ 4ページ ] [ 3ページ ]
見開き3  [ 6ページ ] [ 5ページ ]
```

トンボ・裁ち落とし指定にはChromiumの印刷エンジンを利用しています。商業印刷向けの完全な出力は保証していません。

## 開発

### 必要環境

- Node.js 22.12以降
- npm

macOSでもソースから起動できますが、配布パッケージは現在用意していません。

### ソースから起動

```bash
npm install
npm start
```

### Windows版を作成

インストーラー版とポータブル版を `dist` フォルダへ出力します。

```bash
npm run dist:win
```

### テスト

```bash
npm test
```

ルビ・傍点、章と節、一字下げ、修正マーク、プレビュー同期とページ分割、節の移動、文字コード、状態復元、PDF面付けを自動テストしています。

## 技術構成

- Electron
- JavaScript
- HTML / CSS
- pdf-lib

原稿の保存はElectronのメインプロセス、編集画面と縦書きプレビューはレンダラープロセス、PDFの見開き面付けはpdf-libで処理しています。

## 開発状況

DRFTは開発版です。Windowsを主対象としています。
